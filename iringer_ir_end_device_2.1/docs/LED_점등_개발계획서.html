LED 점등 기능 개발 계획서 (고객사 협의 반영)

대상: iringer_ir_end_device_2.1 | 작성일: 2026-01-30 | 브랜치: feature/blink

1. 개요

고객사 협의를 반영한 LED(PWR LED, GPIO1) 점등 규격 및 구현 계획입니다. Beep(수액 종료 알람) 구간에서는 LED 점등, 비알람 구간에서는 일정 간격 점멸로 동작하며, 데이터 송신 시 LED 점등 기능은 제거합니다.

구현 위치: LED 점등·점멸 로직은 LP Core(Low Power Core)에서 구현합니다. HP Core는 알람 활성 여부 등을 공유 메모리에 기록하고, LP Core가 이를 참조하여 LED를 제어합니다.

3. 상세 요구사항 및 구현 방향

3.1 Beep 알람 울릴 때 LED 점등

요구: Beep(수액 종료 알람)이 울릴 때 LED 점등.
구현 방향 (LP Core): HP Core는 알람 활성 시 공유 메모리(예: lp_core_shared_memory)에 알람 플래그를 설정. LP Core는 주기적으로 해당 플래그를 읽어, 알람이 활성인 동안 LED를 ON 유지. 알람 비활성 시 비알람 구간 동작(일정 간격 점멸)으로 복귀.

관련 코드: LP Core: lp_core_main_lp.c, lp_core_shared_memory.h. HP Core: iringer_ir_end_device_2.1.c에서 알람 조건 체크 후 공유 메모리에 플래그 기록.

3.2 알람이 아닐 때(1이 아닐 경우) 일정 간격으로 점등

요구: 알람이 울리지 않을 때는 일정 간격으로 LED 점등(점멸).
구현 방향 (LP Core): LP Core 내 주기 타이머에서 LED_BLINK_INTERVAL_MS마다 LED를 LED_ON_DURATION_MS만큼 ON 후 OFF. 주기와 점등 시간은 #define으로 제어(설정은 LP Core용 헤더 또는 공유 설정 헤더에 정의).

3.3 점멸 주기 및 점등 시간 #define

항목#define 예시기본값비고
점멸 주기 (간격) LED_BLINK_INTERVAL_MS 60000 (1분) 한 번 점등 후 다음 점등까지의 간격(ms). 상수로 주기 조절 가능.
LED 점등 시간 LED_ON_DURATION_MS 100 (0.1초) 한 번 점등 시 LED가 켜져 있는 시간(ms).

설정 위치 제안: LP Core에서 사용하므로 lp_core/lp_core_algorithm_config.h 또는 iringer_led.h 등에 상수 정의. 테스트/운영에 따라 값만 변경하면 되도록 구현.

3.4 LED 밝기 조절 (PWM 미사용, 1000Hz 루프 활용)

요구: LED 밝기를 #define으로 설정 가능하게 하되, 하드웨어 PWM(LEDC)은 사용하지 않음.
구현 방향 (LP Core): LP Core 메인 루프 주기가 IR 샘플링 기준 약 1000Hz(1ms 주기)이므로, 이 1ms 주기를 하나의 “프레임”으로 활용하여 소프트웨어로 밝기를 조절한다.
  • 원리: 매 1ms(루프 1회)마다 위상 카운터(예: 0~99)를 증가시키고, (위상 % 100) < LED_BRIGHTNESS_PERCENT이면 이번 구간에서 LED ON, 아니면 OFF로 설정한다. 100ms 구간 기준으로 보면 ON 비율이 밝기 퍼센트와 같아져, 체감 밝기가 조절된다.
  • #define: LED_BRIGHTNESS_PERCENT (0~100). 100이면 항상 ON(최대 밝기), 0이면 항상 OFF, 50이면 약 50% 밝기. 설정 위치: lp_core_algorithm_config.h.
  • 적용 시점: “LED를 켜야 하는” 구간(알람 시 상시 ON, 비알람 시 점등 구간 내 ON)에서만 위 밝기 로직 적용. LED를 꺼야 하는 구간(비알람 시 점등 간격 대기)에서는 계속 OFF.
  • 장점: LEDC/타이머 리소스 사용 없음, 기존 1ms 루프만 활용하여 구현 가능.
항목내용
루프 주기약 1000Hz (1ms) — IR 샘플링 주기와 동일
위상 주기100 카운트 (0~99) → 1% 단위 밝기. 필요 시 10 카운트(10% 단위)로 단순화 가능.
조건(phase_counter % 100) < LED_BRIGHTNESS_PERCENT → LED ON, else OFF
#defineLED_BRIGHTNESS_PERCENT (0~100, 기본값 예: 100)

3.5 데이터 송신 시 LED 점등 기능 삭제

요구: 데이터 송신 시 LED 점등 기능 삭제.
삭제 대상:
  • iringer_ir_end_device_2.1.c 내 “리포트 전송 성공 시 GPIO1 LED 1회 깜빡임” 처리: gpio_hold_dis(LED_PWR_GPIO) 후 led_pwr_set(true) → 딜레이 → led_pwr_set(false) 구간 제거 (현재 약 800~806행 부근).
  • Wake-up 시 “리포트 전송 목적”으로만 깨어났을 때 LED를 HIGH로 켜는 처리 제거 (현재 약 2204~2207행 부근: gpio_hold_dis(LED_PWR_GPIO), led_pwr_set(true)).
위 삭제 후에도 Sleep 진입 시 LED OFF·hold, 일반 Wake-up 시 LED 동작은 “알람 시 점등 / 비알람 시 일정 간격 점멸” 규격에 맞게 유지.

4. 구현 작업 목록

LED 점등·점멸 동작은 LP Core에서 구현합니다. HP Core가 하는 일은 (1) 알람 활성 여부를 공유 메모리에 써 두는 것, (2) 예전에 있던 “리포트 전송 성공 시 LED 1회 깜빡임”과 “Wake-up 시 LED HIGH” 코드를 제거한 것뿐입니다.

순서작업파일/위치

6LED 밝기 조절: 1000Hz 루프 활용, PWM 미사용. LED_BRIGHTNESS_PERCENT #define 및 위상 카운터 기반 ON/OFF 비율 적용 (알람·점등 ON 구간에만 적용)LP Core lp_core_main_lp.c, lp_core_algorithm_config.h

5. 동작 요약

  • 구현 담당: LED 점등·점멸·밝기 로직은 LP Core에서 수행. HP Core는 (1) 알람 활성 여부를 공유 메모리에 기록하여 LP가 읽게 하는 것, (2) 예전에 있던 “데이터 송신 시 LED 깜빡임”·“Wake-up 시 LED 켜기” 코드를 제거한 것만 담당.
  • 알람 울릴 때: LED 계속 ON (LP Core가 공유 메모리 알람 플래그 보고 유지). 밝기는 LED_BRIGHTNESS_PERCENT에 따라 1000Hz 루프 위상으로 조절.
  • 알람이 아닐 때: LED_BLINK_INTERVAL_MS(기본 1분)마다 LED_ON_DURATION_MS(기본 0.1초) 동안 점등 후 OFF. 점등 구간에서도 밝기는 LED_BRIGHTNESS_PERCENT로 조절.
  • 밝기: 하드웨어 PWM 미사용. 1000Hz(1ms) 루프에서 위상 카운터(0~99)로 ON 비율 제어 → LED_BRIGHTNESS_PERCENT(0~100) #define으로 설정.
  • 데이터 송신: LED 점등 없음 (기존 송신 시 깜빡임·Wake-up 시 HIGH 제거).
