2. 30분~1시간 구간과의 연관성

흰색 화면이 “30분에서 1시간 정도” 후에 보고되는 점과 맞춰 볼 수 있는 설정·타이머는 다음과 같음.

  • LCD_SLEEP_DELAY: iringer_tft.h에서 LCD_SLEEP_DELAY_SEC = 18000(5시간)으로 정의되어 있으나, 주석에는 “30분 후 LCD OFF: LCD_SLEEP_DELAY_SEC = 1800” 예시가 있음. 과거/다른 빌드에서 1800(30분)을 사용한 경우, 첫 다운링크 수신 후 약 30분 시점에 tft_enter_sleep() 호출 → 백라이트 OFF 및 GPIO HIGH-Z. 이 시점 이후 메인 루프는 계속 print_data()를 호출하는데, print_data()는 tft_is_sleeping을 검사하지 않아 슬립 중에도 SPI/GPIO에 접근 → 불안정한 신호로 패널 상태 오염 → 이후 웨이크 시 흰색/깨진 화면 가능.
  • Phase 2 타이머: BACKLIGHT_ON_DURATION_MS = 300000(5분). 백라이트 ON 조건 만족 후 5분이 지나면 lcd_sleep_timer_callback()에서 tft_enter_sleep() 호출. 30분~1시간은 “5분 타이머가 여러 번 돌거나, LCD_SLEEP_DELAY(30분)와 겹치는” 구간으로 해석 가능.

3. GTT가 측정되다가 0이 될 때 (확인된 재현 조건)

흰색 화면이 GTT가 측정되다가 0 gtt가 될 때 발생한다는 보고가 있음. 해당 구간의 코드 흐름과 가능한 원인을 정리함.

3.1 GTT → 0 시 흐름

  • LP Core: 마지막 방울 감지 후 10초 타임아웃(LP_ALGO_GTT_TIMEOUT_US) 경과 또는 방울 1개만 감지 시 shared->gtt = 0 설정 (lp_core_main_lp.c 602~618행).
  • HP Core calc_drop(): device_data.gtt를 LP 공유 메모리 값으로 갱신 → 0.0f.
  • 알람 조건: device_data.gtt <= 0.0f이면 is_gtt_zero = true → 조건 3(0gtt만) 또는 조건 2(95%+ AND 0gtt)로 알람 조건 충족 (iringer_ir_end_device_2.1.c 346~357행).
  • 알람 활성화 시: tft_set_device_data(&device_data) 후 tft_backlight_turn_on_condition() 호출 (1323~1325행, 1335~1336행). 즉, GTT→0으로 알람이 켜질 때 LCD가 슬립 중이면 웨이크됨.

3.2 GTT→0 시 흰색 화면 가능 원인

  • 슬립 해제 직후 배경 미채움: tft_backlight_turn_on_condition()에서 DISPON 전송 후 print_data()만 호출함. print_data()는 전체 화면을 tft_fillScreen(ST7735_BLACK)으로 초기화하지 않고, 투여량·cc/hr·GTT·신호·배터리 등 UI 요소만 그리므로, 그리지 않는 배경 영역은 그대로 둠. ST7735는 DISPON 후 디스플레이 RAM 내용이 미정의(또는 슬립 전/오염된 값)일 수 있어, 그리지 않은 영역이 흰색으로 보일 수 있음.
  • 정리: GTT→0으로 알람이 울리면서 LCD가 웨이크될 때, DISPON만 보내고 전체 화면을 검정으로 채우지 않은 채 print_data()만 호출하면, 배경이 흰색으로 남는 현상과 일치함.

4. 흰색 화면 발생 가능 시나리오

ST7735는 DISPON 후 RAM에 유효한 데이터가 없으면 기본/미정의 값(흰색에 가까운 경우 많음)을 표시할 수 있음. 따라서 「DISPON을 보냈는데 그 직후 RAM을 채우는 동작이 수행되지 않거나 실패하는 경로」와 「슬립 중(GPIO HIGH-Z)에 그리기 함수가 호출되어 패널에 잘못된 신호가 들어가는 경로」, 그리고 「GTT→0 알람 웨이크 시 배경 미채움」이 흰색 화면의 주요 원인 후보임.

3-4 print_data() early return으로 슬립 해제 직후에도 그리기 생략
force_refresh_after_sleep는 device_data_ptr 체크 다음에만 반영됨. device_data_ptr는 널이 아니지만, 그 다음에 has_changes/need_redraw_icon만으로 early return하는 경우는 force_refresh_after_sleep가 true이면 발생하지 않음. 다만 force_refresh_after_sleep를 false로 만든 뒤 다른 경로에서 print_data()가 호출될 때 early return하면 갱신 누락 가능.		 iringer_tft.c (1968~1970행): force_refresh_after_sleep로 has_changes = true 설정. (2034~2036행): !has_changes && !first_call && !need_redraw_icon이면 return. 슬립 해제 직후 1회 호출에서는 force_refresh로 has_changes가 true이므로 정상적으로 그리기 수행. 단, device_data_ptr가 NULL이면 그 전에 return하므로 4-1과 동일. 중간
4-5 초기화 중 tft_fillScreen(ST7735_BLACK) 실패
tft_init()에서 CASET/RASET/RAMWR까지 보낸 뒤 spi_handle이 NULL이거나 전송 중 실패하면 화면 일부만 검정이거나 미쓰기 상태(흰색)로 남을 수 있음.		 iringer_tft.c (988~1047행): tft_fillScreen() 내부에서 spi_handle NULL 체크, 뮤텍스 실패 시 return. NULL이면 CASET/RASET/RAMWR만 보낸 뒤 return. 전송 실패 시 break로 루프 탈출 → 일부만 채워짐. 중간
3-6 print_data()가 슬립 중(tft_is_sleeping == true)에 호출될 때 GPIO가 HIGH-Z
tft_enter_sleep() 후 CS/DC/RST/MOSI/SCLK는 모두 INPUT(플로팅). 메인 루프는 매 주기(약 1초)마다 tft_set_device_data() 후 print_data()를 호출하며, print_data()는 tft_is_sleeping을 검사하지 않음. 따라서 슬립 중에도 tft_write_cmd/tft_fill_rect_fast 등이 호출되고, GPIO가 HIGH-Z인 상태에서 SPI/GPIO 제어가 이루어져 잘못된 신호가 패널에 들어갈 수 있음. 이로 인해 컨트롤러 상태가 오염되고, 이후 웨이크 시 또는 백라이트만 켜진 상황에서 흰색/깨진 화면으로 보일 수 있음. 30분~1시간 후 보고와 일치하려면 LCD_SLEEP_DELAY=1800(30분) 빌드 또는 Phase 2(5분) 후 슬립 진입이 선행된 경우를 가정하면 됨.		 iringer_tft.c print_data() (1944행~): tft_is_sleeping 검사 없음. iringer_ir_end_device_2.1.c (1424~1426행): 매 주기 tft_set_device_data(&device_data); print_data(); 호출. 높음 (30분~1시간 구간과 연관 가능)
4-7 tft_exit_sleep()만 호출되고 tft_backlight_turn_on_condition()이 호출되지 않는 경로
현재 코드상 tft_exit_sleep()는 tft_backlight_turn_on_condition() 내부에서만 호출됨. 따라서 단독으로 exit_sleep만 호출되어 DISPON 없이 화면만 켜지는 경로는 없음. 만약 추후 다른 경로에서 exit_sleep만 호출하면 GPIO/ST7735 복원 및 DISPON 없이 상태가 꼬여 흰색/깨짐 가능.		 iringer_tft.c (219~243행): tft_exit_sleep()는 tft_is_sleeping = false와 BACKLIGHT hold 해제만 수행. GPIO 복원 및 SLPOUT/NORON/DISPON은 하지 않음. 낮음 (현재 구조상 경로 없음)
4-8 GTT→0 시 알람 웨이크: DISPON 직후 배경 미채움
GTT가 측정되다가 0이 되면 알람 조건(0gtt) 충족 → tft_backlight_turn_on_condition() 호출 → LCD 슬립 해제 시 DISPON 전송 후 print_data()만 호출. print_data()는 전체 화면을 tft_fillScreen(ST7735_BLACK)으로 채우지 않고 UI 요소만 그리므로, 그리지 않은 배경 영역이 ST7735 미정의 RAM(또는 슬립 전/오염 값)으로 인해 흰색으로 보일 수 있음. 현장에서 “GTT가 0이 될 때 흰색 화면”으로 확인된 재현 조건과 일치.		 iringer_ir_end_device_2.1.c (1323~1325행): 알람 활성화 시 tft_set_device_data 후 tft_backlight_turn_on_condition().
iringer_tft.c tft_backlight_turn_on_condition() (694~706행): DISPON 후 print_data()만 호출, 전체 화면 검정 채우기 없음.		 높음 (GTT→0 재현 조건과 일치)

5. 요약 및 권고 사항

가장 유력한 원인 (4-1, 4-2, 4-6, 4-8)
  • 4-1: 슬립 해제 시 tft_backlight_turn_on_condition()에서 DISPON까지 보낸 뒤 print_data()를 호출하는데, device_data_ptr가 NULL이면 print_data()가 아무것도 그리지 않고 return → 흰색 화면.
  • 4-2: 같은 호출에서 print_data()는 실행되지만, 내부의 tft_write_cmd/tft_fill_rect_fast 등이 spi_handle NULL 또는 뮤텍스 실패로 전송을 생략하면, DISPON만 나가고 RAM 쓰기가 안 되어 흰색 화면.
  • 4-6: 슬립 중(tft_is_sleeping == true)에도 메인 루프가 매 주기 print_data()를 호출하는데, print_data()는 tft_is_sleeping을 검사하지 않음. GPIO가 HIGH-Z인 상태에서 SPI/그리기 호출이 이루어져 패널에 잘못된 신호가 들어가고, 이후 웨이크 시 흰색/깨진 화면으로 나타날 수 있음. 30분~1시간 구간 보고와 맞추려면 LCD_SLEEP_DELAY=1800(30분) 빌드 또는 Phase 2(5분) 슬립 후 누적 동작을 가정하면 됨.
  • 4-8 (GTT→0 확인): GTT가 측정되다가 0이 될 때 알람으로 LCD가 웨이크되며, DISPON 직후 전체 화면을 검정으로 채우지 않고 print_data()만 호출하여 배경이 흰색으로 남는 경우. 현장 재현 조건과 일치.

권고 사항

  • 4-1 대응: tft_backlight_turn_on_condition()에서 print_data() 호출 전에 device_data_ptr != NULL을 확인하고, NULL이면 최소한 tft_fillScreen(ST7735_BLACK) 한 번 호출하여 DISPON 후 RAM을 검정으로 채우거나, DISPON 전에 device_data_ptr 설정이 보장되도록 호출 순서/초기화 순서를 정리.
  • 4-2 대응: 슬립 해제·GPIO 복원 직후 1회는 spi_handle 및 뮤텍스 검사 후 실패 시 tft_fillScreen(ST7735_BLACK) 등 폴백으로 전체 화면을 한 번 채우는 경로 추가 검토.
  • 4-6 대응 (필수 권고, 적용 완료): print_data() 진입 직후 if (tft_is_sleeping) return; 추가하여 슬립 중에는 그리기/SPI 접근을 하지 않도록 함.
  • 4-8 대응 (GTT→0 흰색 화면 방지): tft_backlight_turn_on_condition()에서 슬립 해제 시 DISPON 전송 직후 tft_fillScreen(ST7735_BLACK)을 한 번 호출한 뒤 print_data()를 호출하도록 수정. 이렇게 하면 GTT→0 알람 웨이크 시에도 배경이 검정으로 채워져 흰색 화면이 나오지 않음.
  • 4-3 대응: SLPOUT/DISPON 직후 딜레이를 데이터시트 또는 실기 테스트로 확인하고, 필요 시 120ms/100ms를 늘려 적용.
  • 일관성: DISPON을 보내는 모든 경로(초기화 시퀀스, tft_backlight_turn_on_condition())에서 그 직후 반드시 전체 화면 채우기 또는 print_data()로 유효한 내용을 그리도록 보장.
